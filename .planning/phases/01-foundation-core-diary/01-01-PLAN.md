## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Server | Auth credentials, session cookies cross here |
| Server -> Database | All Prisma queries cross here |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Spoofing | Auth login | high | mitigate | Better Auth handles password hashing (Bcrypt default), session tokens stored in httpOnly cookies |
| T-01-02 | Tampering | Auth middleware | medium | mitigate | Session validated on every request via auth.api.getSession(); middleware runs as edge function |
| T-01-03 | Repudiation | Auth registration | low | accept | Audit trail via createdAt timestamps on user records |
| T-01-04 | Information Disclosure | Password reset | low | mitigate | Generic success message ("If account exists, link sent") prevents user enumeration |
| T-01-05 | Denial of Service | Auth endpoints | medium | mitigate | Enable Better Auth built-in rate limiting |
| T-01-06 | Elevation of Privilege | Admin role assignment | high | mitigate | First-user-admin check runs once via afterAuth hook; subsequent registrations always get "user" role; RBAC enforced server-side via admin plugin |
| T-01-SC | Tampering | npm install | high | mitigate | All packages are well-known major libraries (next, prisma, better-auth, tailwindcss, shadcn/ui, react-hook-form, zod, date-fns, next-themes, @tanstack/react-query, lucide-react) |
</threat_model>
